# Размещение Easy Boost рядом с существующим приложением и ботом (VPS в Германии)

Цель: поставить Easy Boost на тот же сервер, **не трогая** текущий сайт и бота кальянной.
Принцип изоляции: своя папка, свой порт, свой процесс PM2, отдельный адрес (поддомен).

## 1. Выбор адреса
- **Поддомен (рекомендуется):** `easyboost.твойдомен.de` — отдельный HTTPS, не мешает текущему сайту. Нужна одна DNS-запись (A на IP сервера, или CNAME на основной домен).
- Отдельный домен — то же, но надо покупать ещё один домен.
- Подпуть `/easyboost` — без DNS, но сложнее в настройке прокси и путей.

## 2. Разведка сервера (безопасно, только читает)
Выполнить по SSH и прислать вывод:
```bash
cat /etc/os-release | head -2
echo "--- Node ---"; node -v 2>/dev/null || echo "Node НЕ установлен"
echo "--- PM2 ---"; pm2 ls 2>/dev/null || echo "PM2 нет"
echo "--- Прокси ---"; which nginx caddy apache2 2>/dev/null || echo "прокси не найден в PATH"
systemctl is-active nginx caddy apache2 2>/dev/null
echo "--- Порты ---"; ss -tlnp 2>/dev/null | grep LISTEN
echo "--- Python (бот) ---"; python3 --version 2>/dev/null
```

## 3. Дальнейшие шаги (после разведки)
1. Если Node нет — поставить Node LTS (Python-бота это не затронет).
2. Положить Easy Boost в отдельную папку `/opt/easyboost`, приложение — в `public/index.html`.
3. Запустить на **свободном порту** (например 3001) через PM2 отдельным процессом:
   `pm2 start server.js --name easyboost` (у бота свой процесс, не пересекаются).
4. В обратном прокси добавить **отдельный блок** для поддомена → проксировать на этот порт.
   Существующие блоки (сайт, бот) не меняем.
5. Выпустить HTTPS для поддомена (certbot для Nginx/Apache; Caddy — автоматически).

## Почему бот и сайт не пострадают
- Разные порты → нет конфликта.
- Отдельный процесс PM2 → падение/перезапуск Easy Boost не влияет на бота.
- Отдельный блок прокси на поддомен → текущие сайты работают как раньше.
- Своя папка и своя база `data.json` → данные изолированы.

Пришли вывод из п.2 + имя домена — соберём точные команды под твой стек.
